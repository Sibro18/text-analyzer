# **Запуск через Docker Compose**

- Шаг 1: Клонируй репозиторий и перейди в проект.
```
git clone https://github.com/your-repo/text-analyzer.git
cd text-analyzer
```
- Шаг 2: Убедись, что Docker и Docker Compose установлены.
```
docker --version
docker-compose --version
```
- Шаг 3: Запусти Docker на своём устройстве.
- Шаг 4: выполни команду `docker-compose up --build` из корня проекта

После запуска сервис будет доступен по адресу: http://localhost:8000

# **Папка tests**

В папке находятся PowerShell‑скрипты, предназначенные для быстрого тестирования API. Каждый скрипт выполняет свою задачу: массовая отправка запросов, проверка статусов и получение результатов.

## **`.\exports.ps1`**

Отправляет несколько параллельных запросов к эндпоинту `public/report/export`.

### **Настраиваемые параметры**

Редактируются в блоке param(...).
```
param(
    [string]$BaseUrl = "http://localhost:8000/public/report/export",
    [string]$BigFile = "big.txt", 
    [string]$SmallFile = "min.txt"
    # etc
)
```
Здесь можно изменить:
- базовый URL,
- файлы для отправки,
- добавить свои параметры.

### **Настройка количества задач**

Чтобы изменить число параллельных задач, редактируется массив $jobs
```
$jobs = @(
    @{ Name = "Big1";  File = $BigFile;   Size = "500MB" },
    @{ Name = "Big2";  File = $BigFile;   Size = "500MB" },
    @{ Name = "Small1"; File = $SmallFile; Size = "small" },
    @{ Name = "Small2"; File = $SmallFile; Size = "small" }
	# etc
)
```
Каждый объект массива — это отдельная задача, отправляемая в API.

## **`.\status.ps1`**

Скрипт проверяет статусы активных задач
Использование:
- передать ID задач вручную
```
.\status.ps1 id1 id2 id3
```

либо внутри скрипта используется файл task_ids.txt, который создаётся после запуска exports.ps1.

## **`.\results.ps1`**

Скрипт получает результаты задач.
Поведение:
- если задача завершена — возвращается файл .xlsx;
- если ещё выполняется — возвращается JSON со статусом.
Использование аналогично status.ps1:
```
.\results.ps1 id1 id2
```
или через task_ids.txt.

# **Возможности**

- Загрузка файлов любого размера
- Морфологическая нормализация слов (pymorphy3)
- Фоновые задачи через Celery + Redis
- Отслеживание статуса задачи
- Результат в формате Excel (.xlsx)
- Автоматическая очистка старых файлов (24 часа)

# **Технологии**

- FastAPI - веб-фреймворк
- Celery - очередь задач
- Redis - брокер сообщений
- SQLite - временное хранилище статистики
- pymorphy3 - морфологический анализ
- openpyxl - генерация Excel
- Docker - контейнеризация
